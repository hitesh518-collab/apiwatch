# Agent Instructions

These instructions apply to the whole repository.

## Implementation Log

- Use `implementation-log/` for local agent status reports.
- The directory is intentionally gitignored. Do not force-add log files.
- Create the directory if it does not exist.
- Write one short Markdown file per meaningful task or work session.
- Keep future entries high-level: goal, decisions, files/areas touched, verification, blockers, and next steps.
- Do not paste exhaustive command transcripts, full diffs, or private/secrets-bearing data.
- The first log entry may be more detailed because it captures the initial project decisions and setup history.
- Before finishing a task, update or create the relevant log entry with the final status.
